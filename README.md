# tsbuild

`tssetup` で構築した Bun + TypeScript プロジェクトにおいて、開発サーバーの起動と TypeScript コンパイラ（`tsc --watch`）による自動ビルド、ブラウザのホットリロードを一元管理する開発支援コマンドです。

## 🚀 特徴

*   **1コマンド起動:** コマンド一つでローカルWebサーバー起動・ブラウザ自動起動・TS監視ビルドのすべてを開始します。
*   **ポート衝突の自動回避:** デフォルトの53000番ポートが埋まっている場合、空いているポートを自動で探し当ててサーバーを起動します。
*   **404エラー防止:** 起動と同時に自動で一度初回のコンパイルを行うため、「JSファイルがまだ生成されておらず読み込めない」といった404エラーを防ぎます。
*   **ホットリロード機能:** `src` 内のコード変更や `index.html` の編集を検知すると、自動的にブラウザをリロードします。
*   **ゾンビプロセスの防止:** `Ctrl + C` で終了した際に、バックグラウンドで起動していたBunサーバー（Job）を自動的に停止・クリーンアップします。

## ⚙️ 動作に必要な環境（依存関係）

このコマンドは、`tssetup` コマンドで構築したフォルダ構成を前提としています。また、システムに以下がインストールされている必要があります。

1.  **PowerShell 5.1 以上** (Windows標準)
2.  **Bun** (高速JavaScriptランタイム)
    *   開発サーバー（`server.ts`）を稼働させるために必要です。
3.  **TypeScript (tsc)** (トランスパイラ)
    *   コマンド内部で `bun x tsc` (初回ビルド) および `bun x tsc --watch` (継続監視) を呼び出すため、プロジェクトの依存関係（あるいはグローバル）に TypeScript が必要です。

## 🛠️ インストール方法

PowerShellで以下のコマンドを実行するだけで、自動的にお使いの環境（`$PROFILE`）に `tsbuild` コマンドがインストールされます。


```powershell
irm https://raw.githubusercontent.com/あなたのユーザー名/tsbuild/main/install.ps1 | iex
```

> [!NOTE]
> * 「あなたのユーザー名」の部分は、GitHubに公開したリポジトリのユーザー名（または組織名）に書き換えてください。
> * インストール完了後、現在のPowerShellセッションに反映するには `. $PROFILE` を実行するか、新しくPowerShellを開き直してください。

## ⚡ 使い方とパラメータ一覧

`tssetup` で構築したプロジェクトフォルダ（`tsconfig.json` や `server.ts` があるルートディレクトリ）に移動して、以下のコマンドを実行します。

```powershell
tsbuild [-Help]
```

### パラメータ（引数）詳細

| パラメータ名 | エイリアス | 型 | 必須 | デフォルト値 | 説明 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **`-Help`** | `-h` | `switch` | いいえ | `$false` | コマンドのヘルプメッセージ（使い方とオプション）を表示して終了します。 |

---

### コマンド実行例

#### 1. 開発環境を起動する
```powershell
tsbuild
```
Webサーバーが立ち上がり、自動ビルドとホットリロードが有効化した状態でブラウザが開きます。

#### 2. ヘルプ情報を表示する
```powershell
tsbuild -Help
```


### コマンド実行時の挙動

1.  **ポート自動探索**: デフォルトでは `53000` 番ポートで開発サーバーを起動しようと試みます。もし既に別の開発でポートが使われていた場合は、自動的に `53001` , `53002` ... と空いているポートを探し出します。
2.  **初回ビルドの実行**: サーバー起動と同時に、バックグラウンドで初回のコンパイル（`tsc`）が行われます。これにより、ブラウザが立ち上がった瞬間に「ビルドファイルがなくて404エラーになる」のを防ぎます。
3.  **ブラウザの自動起動**: ビルドが完了すると、自動的に既定のブラウザで開発サーバーのURL（`http://localhost:53000` など）を開きます。
4.  **リアルタイム変更監視 (tsc --watch)**: コマンドはそのまま待機状態になり、`src/index.ts` などの変更をリアルタイムで監視します。コードを書き換えて保存すると自動的にコンパイルが走ります。
5.  **ブラウザ自動更新 (ホットリロード)**: ファイルが更新されビルドが完了すると、WebSocket通信経由で開いているブラウザにリロード命令が届き、ページが自動でリフレッシュされます。

---

### 終了方法

開発を終了するには、コンソール上で **`Ctrl + C`** を押します。

#### 💡 安全なクリーンアップ設計
`Ctrl + C` が押されたタイミングで、バックグラウンドで動いていたBunのウェブサーバーJob（PowerShell Job）を自動的に検出し、**`Stop-Job` および `Remove-Job` で自動的に停止・消去**します。
これにより、開発サーバーがゾンビプロセスとしてメモリに残り続け、次回起動時にポートを圧迫する問題を完全に防いでいます。
