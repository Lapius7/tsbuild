# tsbuild

`tssetup` で構築した Bun + TypeScript プロジェクトにおいて、開発サーバーの起動と TypeScript コンパイラ（`tsc --watch`）による自動ビルド、ブラウザのホットリロードを1コマンドで一元管理する開発支援ツールです。

## 🚀 特徴

- **1コマンド起動:** コマンド一つでローカルWebサーバー起動・ブラウザ自動起動・TS監視ビルドのすべてを開始します。
- **ポート衝突の自動回避:** デフォルトの53000番ポートが埋まっている場合、空いているポートを自動で探し当ててサーバーを起動します。
- **404エラー防止:** 起動と同時に自動で一度初回のコンパイルを行うため、「JSファイルがまだ生成されておらず読み込めない」といった404エラーを防ぎます。
- **ホットリロード機能:** `src` 内のコード変更や `index.html` の編集を検知すると、自動的にブラウザをリロードします。
- **ゾンビプロセスの防止:** `Ctrl + C` で終了した際に、バックグラウンドで起動していたBunサーバー（Job）を自動的に停止・クリーンアップします。
- **自動バージョン更新:** 実行時に最新バージョンを自動確認し、新しいバージョンがあれば自動で更新します。
- **安全なアンインストール:** `-Uninstall` オプションで確認プロンプト付きの安全なアンインストールができます。

## 🛠️ インストール方法

PowerShellで以下のコマンドを実行するだけで、お使いの環境（`$PROFILE`）に `tsbuild` コマンドがインストールされます。

```powershell
irm https://raw.githubusercontent.com/Lapius7/tsbuild/main/install.ps1 | iex
```

> [!NOTE]
>
> - インストール完了後、現在のPowerShellセッションに反映するには `. $PROFILE` を実行するか、新しくPowerShellを開き直してください。
> - インストーラーは既存の `$PROFILE` を安全に保持したまま `tsbuild` 関数のブロックのみを追記します。既存のユーザー設定は上書きされません。

## ⚙️ 動作に必要な環境（依存関係）

このコマンドは、`tssetup` コマンドで構築したフォルダ構成を前提としています。また、システムに以下がインストールされている必要があります。

1. **PowerShell 5.1 以上** (Windows標準)
2. **Bun** (高速JavaScriptランタイム)
   - 開発サーバー（`server.ts`）を稼働させるために必要です。
3. **TypeScript (tsc)** (トランスパイラ)
   - コマンド内部で `bun x tsc` (初回ビルド) および `bun x tsc --watch` (継続監視) を呼び出すため、プロジェクトの依存関係（あるいはグローバル）に TypeScript が必要です。

## ⚡ 使い方とパラメータ一覧

`tssetup` で構築したプロジェクトフォルダ（`package.json` と `server.ts` があるルートディレクトリ）に移動して実行します。

```powershell
tsbuild [-Help] [-Uninstall]
```

引数なしでプロジェクト外（`package.json` / `server.ts` がないディレクトリ）で実行すると、バージョン情報・開発者情報のインフォ画面が表示されます。

### パラメータ（引数）詳細

| パラメータ名 | 型 | 必須 | デフォルト値 | 説明 |
| :--- | :--- | :--- | :--- | :--- |
| **`-Help`** | `switch` | いいえ | `$false` | コマンドのヘルプメッセージ（使い方とオプション）を表示して終了します。 |
| **`-Uninstall`** | `switch` | いいえ | `$false` | 確認プロンプトの後、`$PROFILE` から `tsbuild` を削除してアンインストールします。 |

---

### コマンド実行例

#### 1. 開発環境を起動する

```powershell
tsbuild
```

Webサーバーが立ち上がり、自動ビルドとホットリロードが有効化した状態でブラウザが開きます。

#### 2. ヘルプ情報を表示する

```powershell
tsbuild -Help
```

## 📁 対象プロジェクトの構成

`tsbuild` は `tssetup` で作成したプロジェクトフォルダ（以下の構成）を前提としています。プロジェクトのルートディレクトリで実行してください。

```
myapp/
├── src/
│   └── index.ts        ← 編集対象のTypeScriptファイル
├── dist/               ← tsc が自動生成するJS出力先
├── index.html          ← フロントエンドのHTML
├── server.ts           ← tsbuild が起動するWebサーバー
├── tsconfig.json       ← TypeScript コンパイラ設定
└── package.json        ← Bun プロジェクト設定
```

## 🚀 プロジェクトの始め方（tssetup との連携）

```powershell
# 1. プロジェクトを作成（カレントディレクトリが自動で移動します）
tssetup myapp

# 2. 開発サーバーを起動
tsbuild
```

---

### コマンド実行時の挙動

1. **ポート自動探索**: デフォルトでは `53000` 番ポートで開発サーバーを起動しようと試みます。既に使われていた場合は `53001`, `53002` ... と自動で探します。
2. **初回ビルドの実行**: サーバー起動と同時に初回コンパイル（`tsc`）を実行します。これにより、ブラウザ起動直後の404エラーを防ぎます。
3. **ブラウザの自動起動**: ビルド完了後、既定のブラウザで開発サーバーのURL（`http://localhost:53000` など）を自動で開きます。
4. **リアルタイム変更監視 (tsc --watch)**: コードを保存すると自動的にコンパイルが走ります。
5. **ブラウザ自動更新 (ホットリロード)**: ビルド完了をWebSocket経由でブラウザに通知し、ページが自動でリフレッシュされます。

---

### 終了方法

**`Ctrl + C`** を押すと、バックグラウンドで動いていたBunサーバーJobを自動的に検出・停止し、ゾンビプロセスが残らないよう安全にクリーンアップします。

---

## 🔄 自動バージョン更新

`tsbuild` はコマンド実行時にGitHubから最新バージョンを自動確認します。新しいバージョンがある場合、自動でインストールスクリプトを取得して `$PROFILE` を更新します。手動で更新したい場合はインストールコマンドを再実行してください。

```powershell
irm https://raw.githubusercontent.com/Lapius7/tsbuild/main/install.ps1 | iex
```

---

## 🗑️ アンインストール方法

`tsbuild -Uninstall` を実行すると確認プロンプトが表示され、`y` を入力すると `$PROFILE` から `tsbuild` 関数が削除されます。

```powershell
tsbuild -Uninstall
```

```
⚠ tsbuild をアンインストールします。本当によろしいですか？ (y/N): y
✅ tsbuild をアンインストールしました。
```

アンインストールは現在のPowerShellセッション内でも即時反映されます。
